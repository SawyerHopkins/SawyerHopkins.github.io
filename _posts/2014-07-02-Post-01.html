Migration Summary

I have migrated my website from some rather old HTML5 template to a new sleek website built upon the Twitter Bootstrap Carousel example. This update should provide a much better presence across all platforms and also help facilitate my new programming and technology blog. I am using Jekyll for the blog as it is the content management system already being used on GitHub Pages. Since most of my work as of late has been with Visual Studio, I opted to install Jekyll on Windows. There are plenty of guides already floating about cyberspace detailing the installation and setup process for every operating system so I won't be writing a step by step tutorial. Instead I will just note a few "optional" steps not specifically mentioned most guides.

This example leads way into the next problem I had setting up Jekyll. The code above is highlighted using Pygments, a python based syntax highlighter. This script puts CSS class tags around keywords depending on the selected language, and as such an appropriate CSS template is needed to actually create the highlighting. The Jekyll webpage suggests a GitHub repo with a syntax.css file, however this file was not complete for the new Pygments 6.0+. Luckily Pygments has a command line function to output a standard CSS file for the current version.

{% highlight console %} $ pygmentize -S default -f html > style.css {% endhighlight %}

This helpful function will spit out a pre-populated style.css file in your current directory which is just about ready for use. The only thing missing was some formatting on the line numbering, which was easily fixed by adding the lineno class to the syntax.css file.

{% highlight css %} .lineno { padding-right: 5px; border-right: 1px solid #ccc; } {% endhighlight %}
